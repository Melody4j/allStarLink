# AllStarLink 数仓ETL爬虫使用指南

## 概述

新的ETL爬虫基于数仓三层架构设计，实现了高效的数据处理流水线：
- **ODS层**：原始数据存储快照
- **DWD层**：事件驱动的数据仓库
- **DIM层**：维度数据管理

## 主要特性

### 📊 数据处理能力
- **向量化运算**：使用pandas进行批量数据处理，避免逐行循环
- **数据增强**：自动补全地理信息（洲、国家）
- **智能分类**：基于规则的组织类型识别
- **异常处理**：坐标验证、数据清洗

### 🏗️ 数仓架构
- **ODS全量快照**：每次执行生成带时间戳的数据快照
- **DWD事件追踪**：自动识别状态变化和地理位移事件
- **DIM维度更新**：Upsert机制保证数据一致性

### ⚡ 性能优化
- **批量插入**：使用SQLAlchemy批处理减少数据库交互
- **连接池**：数据库连接复用
- **分块处理**：大数据集分批写入

```bash

```

### 定时执行模式

```bash

python allstarlink_etl_scraper.py --mode schedule

# 自定义30分钟间隔
python allstarlink_etl_scraper.py --mode schedule --interval 30
```

## 核心功能详解

### 1. 数据抽取（Extract）

从两个数据源获取数据：
- **节点列表API**：获取完整的节点信息
- **地图数据API**：获取在线节点的经纬度信息

### 2. 数据转换（Transform）

#### 数据整合
- 以节点列表为基准，左连接地图数据
- 设置`is_active`标志：有经纬度=在线，无经纬度=离线

#### 地理增强
根据经纬度坐标自动推断：
- `continent`：大洲信息
- `country`：国家信息

#### 组织分类
基于`affiliation`字段智能分类：
- **Organization**：包含关键词(club, arc, network等)或域名后缀
- **Personal**：个人用户（包括callsign与affiliation相同的情况）

#### 数据清洗
- 频率字段：去除非数字字符
- 坐标验证：排除(0.0, 0.0)等无效坐标
- 时间转换：Unix时间戳转换为datetime

### 3. 数据加载（Load）

#### ODS层加载
```sql
-- 全量快照写入ods_asl_nodes_snapshot
-- 每次执行生成新的batch_id (YYYYMMDDHHMM格式)
```

#### DWD事件处理
自动检测并记录以下事件：

**状态变更事件**：
- 事件类型：`status_change`
- 属性名：`is_active`
- 触发条件：节点上线/下线状态改变

**地理位移事件**：
- 事件类型：`geo_move`
- 属性名：`coordinates`
- 触发条件：坐标变化超过0.005度（约500米）

#### DIM层更新
使用MySQL的`INSERT ... ON DUPLICATE KEY UPDATE`实现Upsert：
- 新节点：直接插入
- 已存在节点：更新所有字段
- 维护节点等级、移动属性等维度信息

## 数据质量保证

### 异常值处理
- 坐标(0.0, 0.0)视为无效，不触发geo_move事件
- 频率字段严格数字化处理
- 组织类型冲突检测和处理

### 事件去重
- 仅在数据真正变化时生成事件
- 坐标变化阈值设定为0.005度
- 状态变化必须是真实的0/1切换

### 日志记录
- 完整的执行日志
- 异常情况详细记录
- 数据量统计和质量报告

## 监控指标

每次执行输出关键指标：
- 抽取节点总数
- 在线节点数量
- 生成事件数量
- 更新维度记录数
- 执行耗时统计

## 扩展说明

### 地理信息增强
当前使用简化的地理逻辑，生产环境建议：
- 使用geopy库进行精确地理编码
- 集成离线地理数据库
- 实现地理边界精确判断

### 节点指标计算
当前节点等级和移动属性使用简化逻辑，可扩展：
- 基于历史活跃度计算节点等级
- 基于历史轨迹判断移动属性
- 实现更多业务指标

### 性能优化建议
- 大数据量场景下可考虑分布式处理
- 添加增量更新逻辑减少全量处理
- 实现数据分区提升查询性能

## 故障排除

### 常见问题

1. **数据库连接失败**
   - 检查数据库配置参数
   - 确认网络连接正常

2. **API请求超时**
   - 增加timeout参数
   - 检查网络状况

3. **内存不足**
   - 调整chunksize参数
   - 分批处理大数据集

4. **事件重复生成**
   - 检查比较逻辑
   - 确认阈值设置合理

### 日志分析
所有执行日志包含详细的时间戳和批次ID，便于问题排查。
